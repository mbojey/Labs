Lab 1

Objectives

After completing this lab you should be able to:

  • Use the debugger to find errors in your code.
  • See the benefit of writing good tests.

Pre-Lab Exercises - 2 points

Complete this section before beginning the lab activities. This section will be due AT THE TIME YOUR LAB SECTION STARTS.

This lab will focus on writing tests for the code that we use and using the debugger in Eclipse. There are many ways to test programs that we write. So far, the main technique that you have been taught has been to use print statements in your code so that the console is able to display what is happening in the program. Examples of this would be printing the value of a variable before it is used, i.e. i = 7, or having a statement provide an English description of what code is doing at that time, i.e. "Setting the name of the dog to Bitzy.". These print statements are great at letting someone who is running the code know what is happening. Today we will work to get those little wrinkles out of our code.

Lab Activity - 8 points

What you are given:

  • <%= link_to "MyClass.java" %>

What you have to do:

  • Today you are given a class file that an imaginary student has written. The student has taken COSC 111 but has forgotten some things over the holiday. They wrote code that compiles but does not do what is desired. Luckily, this student uses good coding practice and left us lots of good comments about what the code SHOULD be doing. Your job will be to write a class that will actually do what the comments say this class should be doing.
  • [1 point]Start by creating a test class in Eclipse, call it Test.java. In this class have a main method and write tests for all of the methods in MyClass. You should use print statements so that you can see what is happening in each method. Take the output from testing all the methods and put it in a file called WrongOutput.txt. This will be handed in as part of your solutions.
  • [6 points] Now create a new class and write correct versions of all methods from MyClass.java. Correct is defined by doing what the comments say should be happening. You can continue to use print statements and looking at the code to find the errors or you can use your new skills with the debugger to speed up the process. When you think you have fixed everything run your test class on your new class instead of MyClass. Take the output from testing all the methods and put it in a file called CorrectOutput.txt. This will be handed in as part of your solutions.
  • [1 point] After you are done this describe a scenario, either from your experience or from your imagination, where writing test BEFORE writing the code would be beneficial. Put this answer in a file called TestFirst.txt. This will be handed in as part of your solutions.

Conceptual Questions

  • Why do we need to test our code if it does what we want when we run it?
  • What are the advantgaes of writing good tests?
  • How does testing apply to pair programming i.e. the group work that you did in labs last semester?
