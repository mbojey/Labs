Lab Activity - 8 points

What you are given:

  • The <%=link_to "FeedMe"%> java files from last semester

What you have to do:

  • In lecture you have examples of inheritance and seen the advantages it gives, such as not needing to rewrite code. Today we will be making changes to the FeedMe game to take advantage of this fact so that we can make a more robust game. At the end of the lab we will have 2 different classes for 2 different types of fruit.
  • [1 point] Create a new class and call it Fruit. This will be the parent class that you will work with today. Make this an abstract class as it will not have a constructor.
  • [1 point]Make the Apple class extend the Fruit class. This is because an Apple is a Fruit. This is similar to what you have seen in the book with a Horse class extending a Mammal class. Inheritance always represents an "is a" relationship between object types.
  • [2 points]Take all the attributes OUT of the Apple class and put them IN the Fruit class. Make sure that you use the correct visibility modifier. We will need to access the attributes from any child classes. However, the attributes will still need to be treated as private from any non-child or descendant classes.
  • [2 points]Take all the methods OUT of the Apple class and put them IN the Fruit class. Here we will keep the visibility modifiers that the methods had in the Apple class. There is no need to change anything as methods that previously needed to be public will still need to be accessed by other , non-related classes.
  • [2 points]Now all that should be in the Apple class should be a constructor. As this class is very simple, we can see that it is very easy to add new types of Fruit to the game. Create a new class that will be a new type of Fruit. Make this class a child of the Fruit class with the extends keyword and write a constructor for this new class. Make sure to create a new .png file for the new Fruit so a player can tell the difference.
  • Now you're done! In the next couple weeks we will be able to have the two types of Fruit in the game at the same time!

What you should think about during this activity:

  • What are the other advantages of inheritance?
  • What are some possible disadvantages of inheritance?
  • What would happen if we used shadow variables?
  • What are some examples of overridden methods that are advantageous? Hint: All methods have a default toString() method from the Object class
