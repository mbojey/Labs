Lab 4

Objectives

After completing this lab you should be able to:

  • Be comfortable working with polymorphism.
  • Understand the benefits and power of polymorphism.

Pre-Lab Exercises

Complete this section individually before beginning the lab activities. This section will be due AT THE TIME YOUR LAB SECTION STARTS.

Exercise 1
[1 point] In your own words describe what polymorphism is. This should only take a sentence or two.

Exercise 2
[1 point] You have an interface called Sprite in your program. You are therefore able to create a variable of type Sprite i.e.
     Sprite x;
What would be valid types to store in this variable? That is, what could you fill in the blanks below with?
     x = new ________________(________);
Hand in your answers before your lab section starts.

Lab Activity - 8 points

What you are given:

  • Your work from previous labs

What you have to do:

  • Today all of your hard work with interfaces and inheritance will finally pay off. You will be working the concept of polymorphism to allow you to drop different types of fruits with different score values each!
  • [1 point] The first thing you will need to do it to open your Drop class. In these classes we will need to make some changes to the targets object. Currently it is an ArrayList storing Apple objects. We want it to store Fruit objects so that we can have both types of Fruit.
    • [1 point] You will notice that this change will cause many errors in the code. To fix these errors we will need to change the type of objects stored in the ArrayLists in the Board class to Fruit as well. This will cause even more errors.
    • [1 point] To fix these errors we need to update any object type pulled from the ArrayList to Fruit from Apple.
  • Now you can add any type of fruit you want to the ArrayList of fruit to be dropped during the game. Letâ€™s make it so that you get different Fruit everytime.
  • [1 point]In the dropIt() method of the Drop class we will want to use the random variable r to determine whether to add an Apple or a different piece of fruit. Use the mod(%) function and a conditional statement to make this dream a reality.
    • [1 point]Now add a score attribute to the Fruit class. Also write a public method called getScore() that, as expected, returns the value of score.
    • [1 point]Change the constructor of the Apple and other fruit class so that they take a third int parameter that is the number of points the fruit is worth. Set the score attribute to this parameter.
    • [1 point]As you add the Fruit to the ArrayList we need to update to maximum possible score. Create an attribute in the Drop class called possibleScore and store this value there. This variable should be static. Write a getter (accessor) for this attribute call it getPossibleScore().
    • [1 point]Now you will change how we keep track of the score in the Board class and what score we return to the user. To do this change the
           score++
       line in the checkCollision() method to instead add the score of the current Fruit and change the dropper.getMax() call in the paint() method to instead show the possibleScore.

What you should think about during this activity:

  • What advantages does polymorphism give you in this activity?
  • Would what we have done be possible without polymorphism? If yes, would it be easier or harder?
  • Are there any disadvantages to polymorphism? If yes, what are they?
  • In this activity we used polymorphism through inheritance, could we have used polymorphism through interfaces instead?
